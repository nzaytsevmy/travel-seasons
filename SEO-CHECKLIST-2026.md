# SEO Checklist 2026 — traveltribe.ru (гейт 10/10)

Собрано из **первоисточников-канонов**, не из общих блогов:
- **Google — официальное:** [Search Essentials](https://developers.google.com/search/docs/essentials) · [Creating Helpful, Reliable, People-First Content](https://developers.google.com/search/docs/fundamentals/creating-helpful-content) (дистилляция E-E-A-T) · [Search Quality Rater Guidelines, ред. 11.09.2025, 182 стр.](https://guidelines.raterhub.com/searchqualityevaluatorguidelines.pdf) — документ, по которому асессоры реально оценивают качество.
- **Google — утечка ранжирования 2024:** [разбор Mike King / iPullRank](https://ipullrank.com/google-algo-leak) (2596 модулей API) — подтверждены siteAuthority, NavBoost/клик-сигналы, sandbox новых доменов, siteRadius.
- **Яндекс:** офиц. Яндекс.Вебмастер (ИКС, «Качество сайта») · алгоритм **«Тайфун» 2025** (антифрод ПФ встроен в формулу + переоценка коммерч. факторов) · фильтры Баден-Баден/Минусинск/АГС · [разбор факторов 2026](https://alekseichernysh.ru/blog/factory-ranzhirovaniya-2026) · [коммерческие факторы Яндекса](https://vc.ru/seo/2772116-kommercheskie-faktory-ranjirovaniya-yandeksa).
- **AI-поиск (AEO/GEO):** [AEO-чеклист 48 факторов](https://www.airops.com/blog/aeo-audit-checklist) + фреймворки Aleyda Solis (Orainti), Lily Ray (Amsive), Mike King (iPullRank).
- **Стратегия / топикальная авторитетность:** [Topical Authority Map 2026](https://t-ranks.com/semantic-seo/topical-authority-map/) + [topic clusters канон (Search Engine Land)](https://searchengineland.com/guide/topic-clusters).

Сайт = Astro SSG, аудитория RU + глобал → **значимы оба: Яндекс и Google**. Visa-страницы = lite-YMYL (нужен повышенный EEAT). Монетизация = affiliate (нужен erid/маркировка — см. [[feedback_erid_required]]).

## A. Технический фундамент (Google + Яндекс)
- [ ] HTTPS на всех URL, без mixed content
- [ ] Canonical на каждой странице, self-referential, абсолютный URL
- [ ] robots: `index, follow, max-image-preview:large, max-snippet:-1`; noindex только где надо
- [ ] sitemap: только canonical-URL со статусом 200, **реальный lastmod** (не дата сборки), сегментация по типам
- [ ] AI-боты НЕ заблокированы в robots.txt: GPTBot, ClaudeBot, PerplexityBot, Google-Extended (решение осознанное)
- [ ] Краулабельные ссылки (`<a href>`, не JS-onclick), нет orphan-страниц
- [ ] Чистые короткие URL, без параметров мусора
- [ ] Нет 4xx/5xx на внутренних ссылках; битые ссылки = 0

## B. On-page / контент (Google Helpful Content + утечка 2024 + Яндекс)
- [ ] Оригинальная информация/анализ/опыт, а не пересказ источников; глубина > поверхностного обзора (Google «Helpful Content»)
- [ ] **Ключ в начале `<title>`** (≤60) — `titlematchScore` из утечки явно меряет соответствие title↔запрос; H1, первый абзац, alt, анкоры
- [ ] **Критичный контент — в начало страницы** (Mustang усекает документ по токен-лимиту — утечка)
- [ ] Один H1, иерархия H2/H3 = под-вопросы; meta description ≤155, уникальна, с интентом
- [ ] Изображения: descriptive alt, lazy кроме hero, webp (НЕ avif — [[feedback_safari_avif_bug]])
- [ ] Внутренняя перелинковка 3+ контекстных (триангуляция hub↔visa↔seasons↔blog); анкор descriptive, не «тут»
- [ ] **Тематическая узость страницы** — не уводить от темы сайта (`siteRadius` из утечки штрафует off-topic дрейф)
- [ ] ❌ Red flags (Google, прямой запрет): контент «под трафик», масс-генерация, авто-AI без ценности, под word-count, **искусственное обновление даты без правок контента**, ниша только ради трафика

## C. E-E-A-T / авторство (Google QRG ред. 09.2025 + утечка 2024 + Яндекс YMYL)
- [ ] **Trust — приоритет №1 из E-E-A-T** (QRG): достоверность важнее экспертности; Experience (личный опыт) для travel — сильнейший сигнал
- [ ] Видимый автор + дата: byline «Никита Зайцев · @traveltriberu · данные актуальны на DD MMM YYYY»
- [ ] Author/Person schema, **консистентная entity** автора на всех материалах (утечка: Google явно меряет `isAuthor`/совпадение автора-сущности)
- [ ] About-страница отвечает на **Кто / Как / Почему** создан контент (QRG-линза); раскрытие AI-использования если есть
- [ ] **YMYL расширен (QRG 09.2025):** деньги/здоровье + теперь общество/госуслуги/выборы. Visa/деньги-страницы = повышенный EEAT: ссылка на первоисточник (МИД РФ/посольство/банк), дисклеймер актуальности
- [ ] Цифры конкретны и датированы; дата консистентна в title/URL/Schema (утечка: конфликт дат = сигнал низкого качества)
- [ ] Самопроверка QRG: «забукмаркил бы / порекомендовал? уровень печатного справочника?»

## D. Schema / структурированные данные
- [ ] Organization + WebSite + SearchAction (на главной)
- [ ] BreadcrumbList на всех вложенных страницах (3 уровня)
- [ ] Article/BlogPosting (блог) с datePublished + dateModified
- [ ] FAQPage там где есть FAQ-блок (visa, hub, seasons, calculator)
- [ ] Без ошибок в Rich Results Test и Яндекс-валидаторе

## E. AEO / GEO — цитируемость в AI-выдаче (новый слой 2026)
- [ ] Answer-first: прямой ответ ДО контекста, без длинных вступлений
- [ ] Каждый H2 семантически = вопрос, ответ сразу под ним (×2.8 цитируемость)
- [ ] Абзацы 2–4 предложения, простой синтаксис
- [ ] Скан-формат: списки, таблицы, **жирные** термины
- [ ] Свежесть: original + last-updated дата (70% цитируемых AI страниц обновлены <12 мес)
- [ ] Сущности консистентны (бренд TravelTribe, автор) на всех страницах

## F. Яндекс-специфика (часто игнорируют Google-чеклисты)
- [ ] **Поведенческие — главный вес у Яндекса** (с 2011): CTR сниппета, dwell time, низкий возврат к выдаче. Алгоритм **«Тайфун» 2025** встроил антифрод ПФ прямо в формулу — накрутка = подъём фильтра, никогда не накручивать
- [ ] **ИКС** (Индекс качества сайта) растить органикой: полезность, доверие, бренд. Метрика **«Проксима»** непрерывно сканирует авторитетность/качество
- [ ] Брендовые/прямые визиты (Telegram-канал → сайт) и узнаваемость = сильный сигнал Яндекса
- [ ] **Баден-Баден** (текстовая переоптимизация) — не спамить ключами, естественный язык; **Минусинск** — ссылочный спам; **АГС** — малополезные страницы
- [ ] Коммерческие факторы Яндекса: контакты, прозрачность цен, доверие; для affiliate — честная маркировка «Реклама» + erid (38-ФЗ)
- [ ] Без агрессивной рекламы/попапов, перекрывающих контент
- [ ] Ссылочный спам = 0 (биржевые ссылки = риск Минусинска); подключены Яндекс.Вебмастер + GSC, 0 ошибок

## G. Performance / Core Web Vitals (подтверждённый фактор обоих)
- [ ] LCP < 2.5s, INP < 200ms, CLS < 0.1 (mobile в приоритете)
- [ ] hero — eager + fetchpriority=high; остальное lazy
- [ ] Шрифты subset + font-display swap
- [ ] Mobile-friendly: реальная проверка @402 (контент = десктоп, не урезан)

## I. Стратегия — топикальная авторитетность (канон 2026)
Google Core Update март-2026 закрепил: сайт с 20 связанными статьями по теме обыгрывает сайт с одним мегагайдом.
- [ ] **Топикальная карта:** seed-тема → 3–5 кластеров → 6–12 связанных страниц на кластер (pillar → cluster → support: определения/FAQ)
- [ ] Pillar = исчерпывающий хаб с исходящими на все cluster-страницы; cluster узко и глубоко (1500–2500 слов), линк назад на pillar
- [ ] **Покрытие 4 интентов** на тему: информационный / коммерческий / транзакционный / нишевый-контекстный; вопросы из People Also Ask, Reddit, форумов
- [ ] Анти-каннибализация: одна страница = один интент; pillar и cluster не конкурируют
- [ ] Entity SEO: единый Site Knowledge Graph — сущности (страны, визы, автор, бренд) и их связи консистентны во всём графе schema
- [ ] **Каденс важнее всплеска:** 1–2 cluster-страницы/нед стабильно 6 мес > 20 страниц за 2 нед и тишина
- [ ] Освежать pillar каждые 60–90 дней реальными данными (не косметическая смена даты — Google флагует); квартально ревизия слабых cluster
- [ ] Аудит внутренних ссылок каждые 60–90 дней при росте контента

## J. Измерение и валидация (до и после правок)
- [ ] **Schema:** Google [Rich Results Test](https://search.google.com/test/rich-results) + [Schema Validator](https://validator.schema.org/) + Яндекс-валидатор — 0 ошибок; локально парс JSON-LD по `dist/`, резолв всех @id, нет дублей
- [ ] **CWV:** Lighthouse (lab) **И** поле — Google [CrUX](https://pagespeed.web.dev/) / GSC Core Web Vitals report (field data решает в ранжировании, не lab)
- [ ] **GSC:** Indexing, Core Web Vitals, Manual Actions, Links, запросы/CTR по страницам после деплоя
- [ ] **Яндекс.Вебмастер:** индексация, ИКС, «Качество сайта», диагностика фильтров, поисковые запросы
- [ ] Регресс-факт vs pre-existing: при фейлах визуального сьюта — A/B на чистом HEAD прежде чем винить свою правку

## K. Каноны-первоисточники — ключевые сигналы (для трассируемости)
**Утечка Google 2024 (iPullRank, 2596 модулей):** `siteAuthority` существует (вопреки заявлениям Google) · `NavBoost` рероранжирует по `goodClicks`/`badClicks`/`lastLongestClicks` (клик-данные Chrome, «squashing» против накрутки) · ссылки с `FULLY_TRUSTED` главных весомее · `siteRadius` штрафует off-topic · `titlematchScore` (ключ рано в title) · sandbox новых/перекупленных доменов · возраст домена · только последние 20 редиректов учитываются.
**QRG ред. 11.09.2025 (182 стр.):** Trust — приоритет E-E-A-T · YMYL расширен на общество/госуслуги/выборы · добавлены критерии оценки AI Overviews · линза Кто/Как/Почему · раскрытие AI-генерации.
**Яндекс «Тайфун» 2025:** антифрод поведенческих встроен в формулу ранжирования + переоценка коммерческих факторов; ИКС/Проксима — непрерывная оценка качества/авторитетности.
