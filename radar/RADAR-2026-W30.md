## Топ находок

Находок 🔴 P1 на этой неделе нет.

**🟡 P3 — новое: согласование числительных с существительными.**
Один и тот же класс бага в 4 файлах: живое количество (`X.length`) подставляется
в текст **без склонения по правилам русского числительного** (единственное для
чисел на 1 кроме 11; родительный ед. для 2–4 кроме 12–14; родительный мн. для
остального). На главной (`index.astro`) и в блоге (`blog/index.astro`) для этого
уже есть рабочая функция (`plural`/`pl`, учитывает `%10`/`%100`) — тот же паттерн
не был переиспользован в местах ниже:

- `seasons-direction-count-plural-mismatch` — `destinationRows.length`=74 (падает
  на «2-4») в 3 местах на `/seasons/`: сабтайтл под H1 «**74 направлений** · виза,
  бюджет и цены билетов» (`seasons.astro:180`), ItemList JSON-LD `description`
  «**74 направлений** мира с анализом сезонности...» (`:101`, попадает в Schema.org),
  скрытый a11y-caption таблицы (`:237`). `:191` «по 74 направлениям» — корректно
  (дательный после «по», не баг). Скрин: `radar/assets/seasons-direction-count-plural-mismatch.png`.
- `visa-country-count-plural-mismatch` — `byVisa.free.length`=41 (падает на «1»,
  нужно единственное число) на `/visa/`: тело страницы «**41 стран** принимают по
  загранпаспорту» (`visa/index.astro:316`, должно быть «41 страна принимает») и
  FAQ-ответ, который уходит в `FAQPage.acceptedAnswer.text` — «Сейчас **41
  направлений** принимают...» (`:51`, должно быть «41 направление принимает»).
- `cards-country-count-plural-mismatch` — `countries.length`=42 (падает на «2-4»)
  на `/cards/`: «**42 стран**, источники в каждой карточке» (`cards.astro:95`,
  должно быть «42 страны»). Соседняя фраза в том же файле (`:32`, «из 42 стран»)
  — корректна, там родительный после предлога «из».
- `llms-txt-post-count-plural-mismatch` — `posts.length`=51 (падает на «1») в
  генераторе `scripts/gen-llms.mjs:101`: «51 **статей**» вместо «51 статья».
  Хардкод в шаблон-литерале — будет неверно почти при каждой перегенерации
  (верно только когда счётчик оканчивается на 0 или 5–9). Низкая видимость
  (нишевый файл для AI-ботов), но именно они целевая аудитория этого файла.

Не факт-баги (числа верные) — падежное/числовое согласование, класс P3 по
методологии Radar (косметика, use case не ломает). Общий фикс: переиспользовать
готовую функцию согласования из `index.astro`/`blog/index.astro` в этих 4 местах.
