## Cursor Cloud specific instructions

Назначение раздела: долгоживущие, неочевидные нюансы запуска/тестов для будущих облачных агентов.
Установка зависимостей выполняется update-скриптом при старте VM — здесь её дублировать не нужно.

### Что это за проект
- Статический сайт на **Astro 5** (SSG, `output: static`), собирается в `dist/`. **Бэкенда/БД нет.**
- Для dev/build/test **не нужны секреты и внешние сервисы**: цены рейсов закоммичены
  (`src/data/prices-cache.json`), сайт собирается офлайн. `TRAVELPAYOUTS_TOKEN` нужен только
  для обновления цен (`scripts/fetch-prices.mjs`), не для разработки.
- ⚠️ `README.md` и `CONTRIBUTING.md` **устарели** (описывают старую vanilla-HTML версию).
  Источник истины — `package.json` и `astro.config.mjs`, не README.

### Тулчейн
- Пакетный менеджер — **npm** (`package-lock.json`). `npm ci`/`npm install` запускают `postinstall`
  → `patch-package` (патч `patches/astro-broken-links-checker+1.1.0.patch`); запускать из корня репо.
- Node: CI использует **20**; VM работает на Node 22 — Astro 5 совместим, сборка/тесты проходят.

### Запуск / сборка / линт / тест (команды — в `package.json` scripts)
- **Dev-сервер:** `npm run dev` → http://localhost:4321 (для разработки).
- **Build:** `npm run build` — ~1,5–2 мин, ~2326 страниц (замеры 17.07.2026). `prebuild` генерит `llms.txt`.
  Пре-existing хвост: часть HTML не сжимается astro-compress — это не регресс (см. `CLAUDE.md`).
  ⚠️ В `CLAUDE.md` записано «14 файлов», по факту сборки 17.07.2026 их **45** — число в каноне
  устарело, на глаз ему не верить, считать по логу сборки.
- **Lint (CSS):** `npm run check:css` — **неблокирующий** (`|| true`). stylelint спотыкается о
  frontmatter `.astro` (сотни pre-existing CssSyntaxError) — это НЕ реальный гейт, игнорировать.
- **SEO-аудит:** `npm run check:seo` — сканирует `dist/`, поэтому **сначала `npm run build`**.
- **Visual/functional тесты:** `npm run check:visual` (Playwright). Неочевидное:
  - baseURL тестов = **http://localhost:4322** (built preview), а НЕ dev-сервер 4321.
  - Playwright config **НЕ поднимает webServer сам**. Перед тестами подними preview на 4322:
    `npm run preview:test` (делает build+preview), либо после уже готового `dist/`
    просто `npx astro preview --port 4322` в фоне.
  - Нужны браузеры Playwright (chromium + webkit) — ставятся update-скриптом.
  - Часть webkit-mobile снапшотов может быть `skipped` — это норма, не падение.
  - **Скриншотные эталоны есть под обе платформы:** `*-darwin.png` (локальный pre-push на
    маке) и `*-linux.png` (CI на ubuntu), по ~68 файлов. Playwright сам берёт нужный по
    суффиксу платформы. С 2026-07-18 (PR #92) CI-джоба `visual` гоняет **полный**
    `npx playwright test` с пиксельным сравнением — флаг `--grep-invert "— visual"` убран.
    Теперь зелёная джоба «Visual Regression» реально означает, что вёрстка не поехала.
  - ⭐ **Динамические листинги НЕ снапшотим по пикселям** (`dynamic: true` в `PAGES`,
    сейчас это `blog-index`): их featured-карточка, счётчик статей и облако тегов меняются
    на КАЖДЫЙ новый пост → пиксельный снапшот там давал ложный фейл каждую публикацию.
    Структуру ленты держат тесты `no-overflow` / `no-broken-images`. **Следствие:
    добавление блог-поста визуальных эталонов НЕ трогает — publish идёт в один заход, без
    пересъёмки linux-эталонов.** (Раньше каждый пост требовал gen-linux + докоммит; фикс
    2026-07-21 это убрал. Best practice: не снапшотить волатильный контент.)
  - ⭐ **Контентные статьи НЕ снапшотим по пикселям** (`content: true` в `PAGES`, 11 постов).
    Их текст мы намеренно обновляем ради свежести (Яндекс её поощряет), а любой рефреш
    ломал бы эталон и тянул пересъёмку linux: добавление ОДНОГО раздела в Рицу 2026-07-21
    стоило пересъёмки 4 darwin + 4 linux эталонов и ~40 мин. Держат структурные тесты
    `no-overflow` / `no-broken-images` / `no-css-escapes` — они от правок текста не падают.
    **Блоговую вёрстку стерегут две канарейки** (пиксельные; правишь их — пересними эталон):
    `blog-kamchatka` (единственный пост, где есть сразу все блоговые компоненты:
    AffiliateNote + PricingCards + FlightRoutes + TOC) и `blog-novoafon` (прозовые
    markdown-таблицы + AffiliateNote). Заодно из снапшотов ушёл известный флейк `blog-japan`.
    Итог: **рефреш статьи эталоны НЕ трогает** — правь текст и пушь в один заход.
  - Linux-эталоны снимает одноразовая джоба `gen-linux-baselines` (вручную:
    `gh workflow run visual-tests.yml -f gen_linux=true`): прогон 1 пишет `*-linux.png` через
    `--update-snapshots`, прогон 2 проверяет их стабильность. После смены вёрстки или обложек
    linux-эталоны надо переснять этой джобой и закоммитить (иначе `visual` в CI упадёт).
  - ⚠️ Тайм-аут стабилизации `toHaveScreenshot` поднят до 20с (опция вызова в
    `tests/visual.spec.ts`): на linux-раннере webkit не успевал устаканить самые длинные
    `fullPage` (`blog-kamchatka`, `country-hub-turkey`) за дефолтные 5с и падал на «waiting for
    fonts». Строгость сравнения (`maxDiffPixelRatio 0.02` / `threshold 0.2`) не тронута.
- **Git-гейт перед push.** `scripts/pre-push.sh` (build → check:seo → preview → check:visual)
  — это боевой гейт: он реально блокирует push и идёт ~6 минут.
  - `lefthook.yml` — незаполненный пример из коробки, к делу отношения не имеет.
  - Хук подключается через `.git/hooks/pre-push`. Он **не в репозитории** (`.git/` не
    коммитится), поэтому на свежем клоне (в т.ч. на облачной VM) его нет — push уйдёт
    без проверок. На машине Никиты он стоит и работает.
  - ⛔ Отсюда правило: раз гейт может не сработать сам, прогоняй его руками перед PR —
    `npm run build && npm run check:seo && npm run check:visual`. `CLAUDE.md` требует
    зелёный визуал-гейт ДО пуша, и «у меня хук не стоял» не оправдание.
