## ЧАСТЬ 1 — Дожать топ-5 → топ-3 (на существующем контенте)

Цель — страницы, которые уже ранжируются (по GSC поз 5-12), без новых статей.

1. **Найти striking-distance по GSC:** страницы на поз 5-15 с показами но низким CTR.
2. **Freshness:** обновить `updatedDate` + реальные данные 2026 (не косметика даты — правило SEO-CHECKLIST). AI-cited URLs на 25.7% свежее обычных.
3. **Закрыть competitive gaps:** сравнить с топ-3 конкурентами по запросу — чего не хватает (раздел, таблица, число) — добавить.
4. **Dwell-сигналы** (иначе позиция откатится за квартал):
   - Усилить структуру, добавить таблицы/списки (визуальные якоря удержания).
   - Внутренние ссылки на смежные страницы (глубже сессия).
   - (Опц.) короткое видео/инфографика → session ×2.6 — но не в ущерб LCP.
5. **Внутренняя перелинковка НА дожимаемую страницу** с других постов (descriptive-анкор) — передаёт вес.
6. **PAA-расширение:** добрать вопросы из People Also Ask (AlsoAsked/AnswerThePublic) в FAQ.

## ЧАСТЬ 2 — AEO / GEO (попасть в AI-цитирование)

### Главный рычаг — inverted pyramid
Каждая статья и каждый H2 начинается с **прямого ответа 2-4 предложения** до контекста.
AI сканирует верх страницы первым. У нас это `> **Если коротко:**` (статья) — держать.
Под каждым H2 — первое предложение = прямой ответ (жирным, с числом).

### Citation-worthy формат (что AI цитирует чаще)
1. **Comparison tables** — самый цитируемый формат. Минимум 1-2 на pillar (сезоны, цены, варианты).
2. **Bullet-списки** — второй по цитируемости. Списки вместо прозы где можно.
3. **Semantic chunking** — один концепт на секцию. НЕ мешать определения с инструкциями, НЕ хоронить статистику в длинных абзацах.
4. **Citable facts** — конкретные числа в структуре (таблица/список/жирным), не в прозе. «+45 °C в августе», «безвиз 90 дней», «от 18 000 ₽».
5. **Inline references** — ссылки на первоисточники (МИД, посольство, банк) — особенно YMYL. Повышает доверие AI.

### Featured snippet (питает и SERP, и AI Overviews)
- Под H2-вопросом — **прямой ответ 40-60 слов** первым абзацем (paragraph snippet).
- Где список — **нумерованный/маркированный** (list snippet — 30% всех).
- Где сравнение — **таблица** (table snippet — 20%).
- Definition-сниппет (40% всех) — первое предложение = чёткое определение.
- Reading level 8-10 класс для потребительских тем.

### Schema (машиночитаемость для AI)
- Article/BlogPosting + FAQPage + HowTo + BreadcrumbList + Person — уже в каноне.
- FAQPage: AI извлекает Q-A пары напрямую. Держать FAQ ≥10 с прямыми ответами.

### Технические сигналы для AI-движков
- **Bing Webmaster Tools** — submit sitemap (ChatGPT web search использует Bing-индекс).
- **llms.txt / llms-full.txt** — есть, держать актуальными (прямой сигнал AI-краулерам).
- **AI-боты НЕ блокировать** в robots (GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot, Google-Extended) — уже открыты.
- E-E-A-T: named author + bio + visible dates + inline refs (для ChatGPT-цитирования).

## Чек-лист 10/10 перед публикацией / рефрешем страницы

1. TL;DR `**Если коротко:**` — 2-4 предложения прямого ответа с числами в начале.
2. Каждый H2-вопрос → первое предложение = прямой ответ 40-60 слов (жирным с числом).
3. Минимум 1-2 comparison-таблицы (сезоны/цены/варианты).
4. Списки вместо прозы где уместно; один концепт на секцию (semantic chunking).
5. Конкретные числа в структуре, не похоронены в абзацах.
6. FAQ ≥10 с прямыми ответами (FAQPage schema).
7. Inline-ссылки на первоисточники (YMYL обязательно).
8. `updatedDate` + реальные свежие данные (freshness; не косметика).
9. Внутренние ссылки: 3+ на смежные + добор входящих ссылок на эту страницу.
10. Schema валидна (Article+FAQ+HowTo+Breadcrumb+Person), llms.txt обновлён.
