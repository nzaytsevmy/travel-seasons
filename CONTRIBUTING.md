# Contributing to TravelTribe — Сезоны для путешествий

Привет! Это open source проект о путешествиях по всему миру.  
Мы добавляем только то, чего нет на других сайтах — честные данные, актуальные цены, реальный опыт.

---

## Как добавить новый регион

### 1. Fork & Clone
```bash
git clone https://github.com/nzaytsevmy/travel-seasons
```

### 2. Открой `index.html` и найди массив `data`

Добавь объект в нужную группу (`{ group: '...' }`):

```js
{
  region: 'Название региона',
  sub: 'Города / что смотреть',
  visa: 'free',      // free | evisa | required
  budget: 'mid',     // low | mid | high
  r: ['G','G','P','P','G','O','O','O','G','P','G','G'],  // 12 месяцев
  t: [
    'Январь: описание условий',
    'Февраль: ...',
    // ... 12 строк
  ]
}
```

### Коды сезонов `r`
| Код | Значение |
|-----|----------|
| `P` | Пик — идеальные условия |
| `G` | Хорошо — стоит ехать |
| `O` | Нормально — есть нюансы |
| `B` | Плохо — не рекомендуется |
| `X` | Закрыто / нет смысла |

### Бюджет `budget`
| Значение | Недельный бюджет (перелёт + отель + еда) |
|----------|------------------------------------------|
| `low`    | до $1 500 |
| `mid`    | $1 500–3 000 |
| `high`   | от $3 000 |

### Виза `visa`
| Значение | Значение |
|----------|----------|
| `free`     | Без визы для россиян |
| `evisa`    | eVisa — онлайн, без консульства |
| `required` | Виза — нужно посещение консульства или визовый центр |

## Как добавить цены в калькулятор

Найди массив `PRICES` в том же файле и добавь запись:

```js
{ 
  name: 'Название (Города)',
  iata: 'XXX',              // IATA-код ближайшего аэропорта
  flight: [300, 600, 1500], // [бюджет, комфорт, люкс] в USD от Москвы
  hotel:  [30,  80,  250],  // USD/ночь
  food:   [15,  40,  120],  // USD/день
}
```

## Правила

- **Только реальные данные** — личный опыт или проверенные источники (не ChatGPT)
- **Актуальность** — данные на 2026 год
- **Только уникальные фичи** — не копируем то, что уже есть на Tripadvisor, Lonely Planet и т.д.
- **Один PR = одно направление** — не смешивай несколько регионов в один Pull Request
- **Коммиты на английском** — `add: Uzbekistan region data` / `fix: Japan visa info`

---

## Pull Request

1. Создай ветку: `git checkout -b add/uzbekistan`
2. Внеси изменения
3. Открой PR с описанием: что добавил и откуда данные
4. Жди ревью — обычно в течение нескольких дней
